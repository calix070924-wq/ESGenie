# ESGenie 그린워싱 벤치마크 — 라벨링 가이드라인 (v0.1 초안)

> 목적: 평가용 정답 데이터를 **일관되게·방어 가능하게** 만들기 위한 기준.
> 이 문서가 있어야 여러 사람이 라벨해도 흔들리지 않고, 심사에서 "무슨 기준이냐"는
> 질문에 답할 수 있다. 라벨러는 라벨 전 이 문서를 1회 정독한다.

---

## 0. 한눈에 — 라벨러가 할 일

1. 실제 ESG 보고서/DART 공시에서 **문장**을 가져온다(또는 추출된 후보를 받는다).
2. 각 문장에 **label(greenwash/clean) + category + 해당 축 + 근거**를 단다.
3. 애매하면 `uncertain`으로 두고 사유를 적는다(버리지 않는다).
4. test셋 후보는 **두 명이 독립 라벨** → 불일치만 모아 합의한다.

---

## 1. 그린워싱이란 (판정의 대전제)

ESG 공시에서 **실제보다 더 친환경·사회적으로 보이게 만드는 표현**. 학계는 두 기둥으로 본다.

| 기둥 | 정의 | ESGenie 탐지축 |
|---|---|---|
| **Decoupling (탈동조화)** | 말과 실제 성과의 괴리 | D1·D2·D3·D5 (문장 단위) |
| **Selective disclosure (선택적 공시)** | 유리한 것만 공개, 불리한 건 은폐 | D6 (문서 단위) |

> 핵심 원칙: **"근거가 있으면 정상, 근거 없는 인상 조작이면 그린워싱."**
> 표현이 화려해도 같은 문장/보고서에 검증 가능한 정량 근거가 있으면 정상이다.

## 2. 라벨 스키마 (JSONL 한 줄 = 한 케이스)

```json
{
  "id": "EXT-001",
  "sentence": "선도적인 재생에너지 전환으로 재생에너지 비율 31%를 달성했습니다.",
  "company": "○○전자",
  "sector": "전기전자",
  "source": "2024 지속가능경영보고서 p.42",
  "label": "clean",                      // greenwash | clean | uncertain
  "category": "backed_modifier",         // 아래 3절 카테고리
  "axes": ["D2"],                        // 관련 축 D1/D2/D3/D5/D6
  "evidence_ref": "E-4-2=31% (DART)",    // 정답 판단의 근거(있으면)
  "rationale": "수식어가 동일 문장 정량 실적으로 뒷받침됨",
  "labeler": "정민",
  "split": "dev"                         // dev | test  (test는 튜닝에 절대 미사용)
}
```

문서 단위(D6) 케이스는 `sentence` 대신 `report_id` + `disclosed`/`omitted` 항목 목록을 둔다(6절).

---

## 3. 카테고리 정의 + 예시 (문장 단위)

각 카테고리는 **하나의 전형적 패턴**이다. 예시의 핵심은 "근거 유무".

### 3.1 pure_exaggeration — 공허한 과장 → **greenwash** (축 D2)
정량 근거 없는 최상급·비전·다짐.
- 🚩 "미래지향적이고 차세대 에너지 전환을 위해 최선을 다하고 있습니다." (수치 0)
- 🚩 "업계를 선도하는 친환경 기업으로 거듭나겠습니다."

### 3.2 backed_modifier — 근거 있는 수식어 → **clean** (축 D2, 함정)
화려한 수식어지만 **같은 문장에 검증 가능한 정량 실적**이 동반.
- ✅ "획기적인 다양성 정책으로 여성 구성원 비율 24.8%를 기록했습니다." (S-3-1=24.8% 일치)
- ✅ "선도적인 공정 혁신으로 배출 원단위를 전년 대비 18% 줄였습니다." (실측 -18%)
- 주의: 수치가 **증빙과 일치할 때만** clean. 불일치면 numeric_mismatch.

### 3.3 numeric_mismatch — 수치 불일치 → **greenwash** (축 D1)
문장 수치가 증빙(DART/OCR)과 어긋남.
- 🚩 "온실가스를 30% 감축했습니다." (실제 증빙 -5%)
- 구분: 연도·고객번호·페이지 같은 **식별자 숫자는 수치 주장 아님** → clean.

### 3.4 numeric_match — 수치 일치 → **clean** (축 D1)
- ✅ "재생에너지 사용 비율은 42%입니다." (E-4-2=42% 일치)

### 3.5 timeseries_contradiction — 추세 역행 → **greenwash** (축 D5)
증감 주장이 실측 시계열과 반대.
- 🚩 "폐기물 배출을 꾸준히 줄여왔습니다." (실측은 매년 증가)

### 3.6 future_plan — 미래 계획 → **clean** (축 D5, 함정)
목표·계획은 **실적 주장이 아니므로** 그 자체로 그린워싱 아님.
- ✅ "2030년까지 탄소중립 달성을 목표로 합니다." (목표 선언)
- 주의: 목표를 **이미 달성한 실적처럼** 쓰면 greenwash(→ timeseries/numeric).

### 3.7 clean_factual — 단순 사실 → **clean**
- ✅ "사업장은 경기도 수원시에 위치합니다."

> 카테고리가 애매하면 **가장 지배적인 패턴 하나**를 고르고 `axes`에 복수 축을 적는다.

## 4. 판정 결정 트리 (헷갈릴 때 순서대로)

1. 문장에 **검증 가능한 수치**가 있나?
   - 있다 → 증빙과 **일치**? → clean(numeric_match/backed_modifier)
   - 있다 → 증빙과 **불일치**? → greenwash(numeric_mismatch)
   - 숫자가 **식별자/연도**면 수치 주장 아님 → 다음 단계로.
2. **증감·추세** 주장인가? → 실측 방향과 비교(D5). 역행이면 greenwash.
3. **미래 목표/계획**인가? → clean(future_plan). 단, 실적인 척하면 greenwash.
4. **최상급·모호 수식어**가 정량 근거 **없이** 쓰였나? → greenwash(pure_exaggeration).
   근거가 동반되면 → clean(backed_modifier).
5. 위 어디에도 안 걸리는 단순 사실 → clean.

**경계 규칙 (자주 틀림):**
- "근거가 같은 문장/같은 페이지에 있으면" 근거 있는 것으로 본다.
- 부분적 근거(일부만 뒷받침) → `uncertain` + 사유.
- 정성 표현이라도 **검증 가능한 구체 행위**(인증번호, 제3자 검증 명시)면 clean 쪽.

## 4.5 규제기관 4대 판단 기준 (환경부·공정위 실무 기준)

제품·서비스 친환경 주장은 한국 규제기관이 아래 4가지를 본다. 라벨 시 동일 기준 적용:

1. **객관적 근거가 있나** — 시험성적서·인증서·LCA(전과정평가)·함량자료 등. 근거 없으면 greenwash 쪽.
2. **제품 전체 주장인지 일부 주장인지** — 일부 부품·원료만 친환경인데 전체처럼 광고하면 위험(→ partial_truth, D6).
3. **전제조건을 충분히 적었나** — "산업적 퇴비화 시설에서만 분해", "특정 조건에서만 생분해" 등 조건 누락 시 위험(→ condition_omitted).
4. **소비자가 어떻게 받아들이나** — 사업자 의도보다 **소비자 오인 가능성**이 핵심.

### 규제기관 적발 그린워싱 유형 (제품·광고 문구 라벨용)

| 유형 | 정의 | 예시 표현 | 축 |
|---|---|---|---|
| **absolute_unsubstantiated** | 절대형·검증불가 단정(근거 없음) | "탄소중립", "무공해", "무해", "100% 생분해", "완전 분해" | D2 |
| **condition_omitted** | 전제조건 누락 | "생분해", "퇴비화 가능", "재활용 가능"(조건 없이) | D2 |
| **partial_truth** | 일부 주장을 전체처럼 | "리사이클 의류"(일부 원단만), "친환경 수세미"(일부 소재만) | D2·D6 |
| **vague_abstract** | 막연·검증불가 추상 | "지구를 위한", "자연친화적", "그린", "에코" | D2 |
| **vague_label** | 실질 근거 빈약한 라벨 | "ESG 펀드", "친환경 항공권"(상쇄·캠페인 수준) | D2 |

> 이 유형들은 `esgenie/knowledge/greenwash_lexicon.py`의 `ABSOLUTE_UNVERIFIABLE` /
> `CONDITION_REQUIRED` / `VAGUE_ABSTRACT`로 1차 매칭된다(LLM 2차 판정이 근거·조건 유무로 확정).

## 5. 골드 라벨 출처 (사람 판단 부담을 줄이는 지름길)

규제기관이 이미 판정한 사례는 **라벨이 공짜로 신뢰**된다. 우선 활용:
- **공정거래위원회** 친환경 위장·부당 표시광고 제재 사례
- **환경부 / 한국환경산업기술원** 친환경 표시·인증 위반 사례
- 해외: FTC Green Guides, EU Greenwashing Directive 적발 사례(번역 참고)
- 학계 공개 데이터셋(예: 그린워싱 분류 논문 부록)

수집 시 `source`에 출처와 판정기관을 명시한다(예: "공정위 2023-○○호").

## 6. D6 문서 단위 라벨링 (선택적 공시)

문장이 아니라 **보고서 1건**을 단위로 라벨한다.

```json
{
  "report_id": "RPT-007", "company": "○○화학", "sector": "화학",
  "label": "greenwash",                 // 선택적 공시 의심 여부
  "d6_type": "orphan_ratio",            // orphan_ratio | sensitive_omission | both | none
  "disclosed_favorable": ["E-6-2"],     // 자랑한 유리 비율
  "omitted_context": ["E-6-1"],         // 누락한 분모/불리 항목
  "rationale": "재활용률은 강조, 폐기물 총량은 미공시",
  "split": "dev"
}
```

판정 기준:
- **orphan_ratio**: 유리한 *비율*을 공시했는데 그 *분모/총량*(불리)을 누락 → greenwash 강신호.
- **sensitive_omission**: 배출·폐기물·오염·법규위반·산재 등 **불리 항목을 통째로 누락**.
- 정상(none): 불리 항목도 함께 공시하거나, 애초에 해당 사업 특성상 없는 항목.
- 주의: 단순히 커버리지가 낮은 것(영세기업)과 **의도적 은폐**를 구분 — 유리한 건 강조하면서
  짝이 되는 불리 항목만 빠졌을 때가 진짜 신호다.

## 7. 데이터 수집 가이드 (목표 규모·구성)

- **규모**: 1차 목표 150건(문장) + D6 30건(보고서). 가능하면 300건.
- **다기업·다업종**: 한 기업이 전체의 30%를 넘지 않게. 최소 5~8개 업종.
- **층화(stratify)**: 7개 카테고리가 각각 최소 10건 이상. clean:greenwash ≈ 1:1.
- **분리(split)**: 수집 후 dev:test = 7:3 고정. **test는 라벨 확정 후 봉인** — 프롬프트·임계값 튜닝에 절대 쓰지 않는다.

---

## 8. 품질 관리

- **이중 라벨(test셋)**: 2명 독립 라벨 → 불일치 케이스만 모아 합의. 합의 불가 → 제외.
- **일치도 기록**: 불일치율을 적어두면 "라벨 신뢰도"의 근거가 된다.
- **라벨 누설 금지**: test셋 문장을 few-shot 예시나 프롬프트에 넣지 않는다.
- 버전 관리: 케이스 추가/수정 시 `id` 유지, 변경 이력은 git으로.
