# 증빙 부록(원본 페이지+bbox) 라이브 검증 체크리스트

작업 범위: PDF 실사 응답서에 원본 증빙 페이지 이미지+주황 bbox 박스를 임베드.
샌드박스에서 코드·정합·E2E는 통과(307 passed). 아래는 **로컬 실제 화면**에서만 확인 가능한 것들.

---

## 0. 준비
- [ ] `streamlit run app.py` 정상 기동 (plotly/streamlit 로컬에 설치돼 있어야 함)
- [ ] 한글 폰트 임베드 확인 — PDF 글자가 안 깨짐 (깨지면 `ESGENIE_PDF_FONT` 설정 필요)

## 1. 업로드 → 검출 → 응답서
- [ ] `시연증빙세트_한울정밀공업/` 의 증빙 업로드 (전기요금·도시가스·폐기물명세 최소 3종)
- [ ] 검출 실행 → '📤 실사 응답서' 탭 진입
- [ ] 양식 셀렉터 기본값 = `kesg28`, 응답표가 채워짐

## 2. PDF 다운로드 & 핵심 확인 ⭐
- [ ] '📄 실사 응답서 (.pdf)' 버튼으로 PDF 받기 (버튼이 에러 없이 뜸)
- [ ] PDF 마지막에 **"증빙 부록 — 원본 대조"** 섹션이 있다
- [ ] 부록에 **원본 고지서 페이지 이미지**가 실제로 박혀 있다 (텍스트만 아님)
- [ ] 각 이미지에 **주황색 박스**가 그려져 있다
- [ ] ⭐ **박스가 엉뚱한 데가 아니라 해당 수치/항목 근처**에 찍혀 있다
      → 어긋나면: OCR 노드 bbox 품질 문제(좌표계는 검증됨). 어느 문서가 틀어지는지 메모
- [ ] 응답표 '근거' 칸에 **`→ [E1]` 같은 상호참조**가 보이고, 부록의 `[E1]`과 대응된다

## 3. 누락 방어 (안 죽는지)
- [ ] 증빙 없는 문항(작성필요/증빙대기)이 섞여 있어도 PDF가 정상 생성된다
- [ ] 그 문항들은 부록에 안 나오고, 본문 표/체크리스트는 멀쩡하다

## 4. 파일 흔적 확인 (선택)
- [ ] `outputs/_supplychain/kesg28/evidence_pack/` 에 원본이 복사돼 있다
      → 비어 있으면: 업로드 파일명 ↔ OCR `source_file` 불일치 가능성 (알려줘, basename 매칭 보강함)
