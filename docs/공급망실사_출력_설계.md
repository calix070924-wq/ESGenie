# 공급망 실사 응답 출력 모듈 — 설계 문서

> ESGenie의 사업화 1순위 산출물. 검출 엔진(D1~D6) + OCR 증빙 + provenance를
> "대기업이 받아쓰는 협력사 ESG 실사 응답서"로 변환하는 모듈.
> 작성 2026-06-18 · 1차 버티컬 = 자동차부품 · 본선 시연영상 핵심 장면 후보.

---

## 0. 한 줄 정의

**증빙(전기·가스·폐기물 고지서 + DART + 사내규정)을 넣으면 → 대기업 공급망 ESG 자가진단 양식이 자동으로 채워지고, 각 답변에 원본 증빙 위치가 박히고, 과장·누락이면 깃발이 선다.**

다른 SW는 빈 설문지를 준다. ESGenie는 *증빙에서 자동으로, 검증된 채로, 모든 고객사 양식으로 한 번에* 채운다.

---

## 1. 왜 이게 돈 내는 지점인가 (시장 근거)

현재 ESGenie는 "그린워싱을 잘 잡는 엔진"이다. 하지만 협력사·대기업이 **돈을 내는 결과물**은 검출 점수가 아니라 *제출 가능한 실사 응답서*다. 그 변환 레이어가 비어 있었다 (메모리: PR3로 계속 미뤄진 출력양식 매핑).

리서치로 확인된 실제 시장 구조:

| 사실 | 출처 | 시사점 |
|---|---|---|
| 자동차 산업 표준 자가진단 **SAQ 5.0** (Drive Sustainability) — 7개 섹션, OEM 13社 공동 사용 | drivesustainability.org | 양식이 이미 표준화·고정됨 → 매핑 대상이 명확 |
| 현대차 협력사 ESG 평가 = **6단계** (자가진단→근거자료 제시→서면→현장실사→개선계획→모니터링), 1차 협력사 1,454社 | 현대트랜시스 | "근거자료 제시"가 필수 단계 = 증빙 그라운딩이 핵심 자격 |
| K-Mobility 브릿지 재단(구 KAP) 공급망 자가진단 = 환경 47 + 사회 41 등 88지표 / K-ESG 공급망 가이드 = 4영역·18범주·60항목 | k-esg.org, kmbridge | 한국 자동차부품 특화 양식 존재 → 버티컬 정합 |
| SAQ 거의 모든 문항이 **"Please upload relevant document"** 증빙 첨부 요구 | SAQ 5.0 원문 | ESGenie OCR+provenance가 정확히 메우는 빈칸 |

**협력사의 진짜 고통:** 같은 자가진단을 OEM별로(현대·기아·볼보…) 반복 작성 + 근거자료 수기 첨부 + 과장하면 현장실사에서 감점. **ESGenie가 한 번 증빙 넣으면 모든 양식으로 출력**하면 이 고통이 사라진다 → 그게 B2B2B의 "B" 둘을 다 만족시킨다:

- **중소 협력사(사용자):** 반복작성 제거 + 과장 리스크 차단 + 부족 증빙 안내
- **대기업 OEM(가치 수혜자):** 표준화·증빙연결·그린워싱검증된 응답 수신 → 실사 비용↓ 신뢰도↑
- **ESGenie:** 양식은 선언적 설정이라 OEM/산업 추가가 곧 매출 확장

## 2. 아키텍처

기존에 합의된 **"선언적 설정 + 공통 엔진"** 패턴(industry/ 모듈)과 동일한 철학으로 신설.

```
esgenie/supplychain/                  ← 신설 서브패키지
  __init__.py
  schema.py        # Framework / Question / ResponseSheet / Answer 데이터모델
  frameworks/      # 양식 = 선언적 설정 (코드 아님)
    saq5.py        #   Drive Sustainability SAQ 5.0
    hyundai.py     #   현대차 협력사 자가진단
    kap.py         #   K-Mobility(KAP) 60항목
  mapping.py       # K-ESG 항목 / SSOT metric → Question 답변 도출 규칙
  responder.py     # 엔진: (SSOT graph + 검출결과 + Framework) → ResponseSheet
  gating.py        # D1/D6/D2 검출결과 → 신뢰 배지·HITL 에스컬레이션
  exporters/
    excel.py       # ResponseSheet → .xlsx (OEM 제출본)
    pdf.py         # ResponseSheet → .pdf (증빙 이미지 임베드, pdf_render 재사용)
```

### 데이터 흐름 (기존 파이프라인에 1단 추가)

```
DART + OCR 증빙 ─► L0 SSOT ─► L1 K-ESG추출 ─► L3 검출(D1~D6) ─► L5 audit_trace
                                                   │
                                                   ▼
                              ┌──────────────────────────────────────┐
                              │  supplychain.responder               │  ← NEW
                              │  Framework(선택) + SSOT + 검출결과     │
                              │     → 각 문항 answer 도출              │
                              │     → evidence_node_ids 연결(provenance)│
                              │     → gating으로 신뢰배지/플래그       │
                              └──────────────────────────────────────┘
                                                   │
                                       ┌───────────┴───────────┐
                                       ▼                       ▼
                                 excel.py(.xlsx)         pdf.py(.pdf)
                                 OEM 제출본              증빙 임베드본
```

핵심: **새 검출 로직을 만들지 않는다.** 이미 있는 L1 추출·provenance·D1·D6를 *재조립*해 양식 칸에 떨군다. 그래서 구현 리스크가 낮고, 시연 임팩트는 높다.

## 3. 핵심 데이터 모델 (schema.py)

```python
QType = Literal["yes_no", "multi_select", "numeric", "text", "yes_no_evidence"]

@dataclass(frozen=True)
class Question:
    qid: str                      # "SAQ-E-10", "SAQ-E-10a-GHG" 등
    section: str                  # "Environment"
    text: str                     # 원문 문항
    qtype: QType
    options: tuple[str, ...] = () # multi_select 보기
    evidence_required: bool = False
    # ── 매핑 (이 양식 칸을 무엇으로 답하는가) ──
    kesg_items: tuple[str, ...] = ()      # 예 ("E-3",) 온실가스 배출량
    ssot_metrics: tuple[str, ...] = ()    # 예 ("electricity_kwh",)
    derive: str | None = None             # 도출 규칙 키 (mapping.py)

@dataclass(frozen=True)
class Framework:
    key: str                      # "saq5" | "hyundai" | "kap"
    label: str
    questions: tuple[Question, ...]

@dataclass
class Answer:
    qid: str
    value: object                 # bool / list[str] / float / str
    status: Literal["verified", "self_reported", "insufficient", "flagged"]
    evidence_links: list[EvidenceLink]   # 기존 audit_trace.EvidenceLink 재사용
    flags: list[str]              # ["D6:폐기물총량 누락", "D2:과장수식어"]
    rationale: str

@dataclass
class ResponseSheet:
    framework_key: str
    corp_name: str
    answers: list[Answer]
    coverage_pct: float           # 자동응답 가능 비율
    gaps: list[str]               # "이 항목을 채우려면 X 증빙 업로드"
```

## 4. 매핑 규칙 (mapping.py) — 설계의 심장

각 문항을 **무엇으로 어떻게 답하는가**. 3가지 도출 유형:

| 유형 | 입력 | 답변 도출 | 예시 |
|---|---|---|---|
| **존재형** | K-ESG 항목/사내규정 TextNode 존재 여부 | 있으면 Yes + 해당 증빙 첨부 | SAQ Q10 환경방침 → 사내규정집에 환경방침 조항 있음 → Yes + 규정집 p.X |
| **수치형** | SSOT numeric metric (OCR 증빙에서 추출) | 값 직접 기입 + bbox 증빙 | 전력사용량 128,400 kWh → 에너지/배출 문항 값 |
| **체크형** | 여러 K-ESG 항목 커버리지 | 충족 항목만 체크 | SAQ Q10a → {GHG보고, 에너지효율, 폐기물감축} 중 증빙 있는 것만 tick |

**도출 시 반드시 거치는 게이트 (gating.py):**

1. **D1 수치 정확성** — 답변 값 ↔ 증빙 값 불일치 → `status=flagged`, 자동제출 금지, HITL.
2. **D6 선택적 공시** — 양식이 요구하는 민감항목(배출·폐기물·산재·법규위반)이 SSOT에 없음 → 빈칸을 *조용히 비우지 않고* `status=insufficient` + gap 안내 ("재활용률만 있고 폐기물 총량 없음 — 실사 감점 위험").
3. **D2/D3 과장** — 정성 답변(text)에 과장 수식어/검증 안 된 환경속성 주장 → `flag` + 순화 제안.

신뢰 배지(출력본에 표기):

| 배지 | status | 의미 |
|---|---|---|
| ✅ 증빙검증 | verified | 원본 증빙 + bbox로 D1 통과 |
| ⚠️ 자가신고 | self_reported | 증빙 없이 정성 응답(허용되나 표시) |
| ❗ 데이터부족 | insufficient | 양식 요구하나 증빙 없음 → 보완 안내 |
| 🚩 검토필요 | flagged | D1 불일치 / D2 과장 / D6 누락 의심 |

## 5. 샘플 출력 목업 — SAQ 5.0 Environment (자동차부품 협력사 "한국정밀")

> 시연영상에서 화면에 띄울 결과물. 증빙은 전기요금 고지서·폐기물 위탁처리 명세서·사내 환경방침서.

| 문항 | 자동 답변 | 신뢰 | 근거 (provenance) |
|---|---|---|---|
| **Q10** 공식 환경방침 보유? | **Yes** | ✅ 증빙검증 | 사내 환경경영방침서 v2.1, p.1 §2 "환경 법규 준수 및 지속적 개선" |
| **Q10a** 방침 적용 영역 | ☑ GHG보고 ☑ 에너지효율 ☑ 폐기물감축 ☑ 재사용·재활용 ☐ 수자원 ☐ 생물다양성 | ✅ / ❗ | 체크된 4종 = 증빙 매칭. 수자원·생물다양성 = 증빙 없음(보완 안내) |
| **Q10b** 임직원 환경교육 실시? | **Yes** | ⚠️ 자가신고 | 교육일지 미업로드 — 증빙 추가 시 ✅ 승격 |
| **Q11** 환경경영시스템(ISO 14001 등) | **Yes — ISO 14001** | ✅ 증빙검증 | 인증서 스캔, 유효기간 2025–2028 |
| **(수치) 연간 전력사용량** | **128,400 kWh** | ✅ 증빙검증 | 한전 전기요금 고지서 2025-03, p.1 bbox[0.084, 0.234] "사용전력량 128,400" |
| **(수치) 폐기물 재활용률** | **92%** | 🚩 검토필요 | **D6 선택적 공시: 재활용률만 공시, 폐기물 총량 누락. 분모 없는 비율 → 실사 감점 위험. 폐기물 위탁처리 명세서 업로드 권장.** |

**커버리지: 78% 자동 응답** · 보완 필요 3건(임직원 교육일지, 수자원 사용량, 폐기물 총량)

이 한 장이 시연의 절정이다 — ✅가 줄줄이 채워지다가 마지막 폐기물 행에서 🚩가 뜨고 "이대로 제출하면 감점"이 뜨는 순간이 임팩트.

## 6. 시연영상 시나리오 (60~90초)

1. **(문제)** 협력사 책상에 종이 고지서 더미 + "현대차·기아·볼보가 각각 보낸 ESG 설문 3종" → 막막함. 자막: "협력사는 같은 답을 양식마다 다시 쓴다."
2. **(투입)** ESGenie에 전기·가스·폐기물 고지서 + DART + 사내규정 드래그 업로드.
3. **(마법)** SAQ 5.0 환경 섹션이 자동으로 채워짐. 각 답변 옆 원본 고지서 썸네일에 bbox 하이라이트 박스가 "사용전력량 128,400 kWh"에 정확히 박힘.
4. **(차별점)** 폐기물 행에서 🚩 — "재활용률 92%만 공시·총량 누락 = D6 선택적 공시 위험. 이대로 제출 시 실사 감점." 보완 안내 표시.
5. **(확장)** 한 클릭 → 현대차 양식 / Drive SAQ / KAP 양식으로 **동시 출력** (Excel·PDF 다운로드 애니메이션).
6. **(카피)** "증빙에서 자동으로, 검증된 채로, 모든 고객사 양식으로 한 번에. — ESGenie"

decoupling(D1·D2·D5)·selective disclosure(D6) 검출이 *따로 노는 점수*가 아니라 *출력물을 신뢰하게 만드는 근거*로 합쳐지는 게 핵심 서사.

## 7. 빌드 계획 (PR 분할 — 구조 먼저 → 데이터 → 사업화)

기존 작업 패턴(industry 모듈, held-out 검증)과 동일하게 골격→데이터→캘리브레이션.

**PR1 — seam + 시연 가능 최소본 (본선 영상용, 우선)**
- `supplychain/schema.py` 데이터모델 + `frameworks/saq5.py`에 **환경 섹션 6문항만** 선언
- `responder.py` 엔진 + `mapping.py` 3유형 도출 + `gating.py`(D1·D6 연결)
- `exporters/excel.py` 1종
- 기존 L1 추출·provenance·detect 결과를 재사용(새 검출 없음)
- 테스트: ① 증빙 없음 → 전부 `insufficient`(회귀가드) ② 샘플 DART+OCR → 목업대로 응답 채워짐 ③ D6 누락 케이스 → 폐기물 행 `flagged`
- **이거면 5번 목업 = 영상 절정 장면을 실제로 찍을 수 있다.**

**PR2 — 양식 확장 + 증빙 임베드 PDF**
- SAQ 전체 7섹션 + `frameworks/hyundai.py` + `frameworks/kap.py`
- 커버리지·gap 리포트 UI (app.py 새 탭 "실사 응답서")
- `exporters/pdf.py` — 답변 옆에 원본 페이지 이미지+bbox 박스 임베드(pdf_render.py 재사용)

**PR3 — 사업화 (나중)**
- 다중 OEM 동시 출력 / 변경분(전년 대비) 관리 / OEM 포털 제출 연동 / CBAM 양식 추가(범용 매핑 레이어로 일반화)

**시연 기준으로는 PR1만 완성하면 충분.** 과빌드 금지 — 환경 섹션 한 양식으로 절정 장면을 확보하고, 나머지는 영상에 "확장 가능" 한 컷으로만.

## 8. 열린 질문 / 다음 결정

- **양식 우선순위:** 시연 임팩트는 SAQ 5.0(국제표준·시각적)이 1순위, 현대차 자가진단은 "국내 OEM 직결" 메시지로 2순위. KAP는 한국 자동차부품 특화 보강. → PR1은 SAQ 환경 섹션으로.
- **샘플 증빙 확보:** 목업을 실제로 찍으려면 한국정밀 같은 가상 협력사용 증빙 세트(전기 고지서·폐기물 명세서·환경방침서)가 필요. data/sample_dart에 이미 SME 샘플 있음(hankook_precision) → 여기에 OCR 증빙 PDF 몇 장 추가하면 됨.
- **수치 도출 신뢰성:** 전력 kWh → 배출량(tCO2e) 환산은 industry 모듈의 emission_factors를 타야 함 → 자동차부품 배출계수(현재 빈 스켈레톤) 일부라도 채워야 수치 문항이 산다.
