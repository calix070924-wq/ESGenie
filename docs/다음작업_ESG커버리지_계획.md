# 다음 작업 계획 — ESG 전 항목 커버리지 (K-ESG 28 기반)

> 2026-06-18 브레인스토밍 합의. 다음 세션에 이 문서부터 열고 바로 시작.

## 합의한 방향 (왜 이렇게 가는가)

**1. 기반 = K-ESG 28 (캐노니컬), 폼 = 어댑터.**
K-ESG 28을 내부 표준(substrate)으로 둔다. SAQ 5.0 / 현대차 / KAP 폼은 그 위에 얹는 출력 어댑터(이미 `frameworks/`가 어댑터 패턴). 폼은 "대체"가 아니라 "추가". K-ESG는 substrate지 고객 제출물이 아니므로, 최종 산출물(폼 응답) 능력은 절대 버리지 않는다.

**2. S·G 버리지 않는다. 영역이 아니라 데이터 타입으로 자른다.**
자동화 가능 여부는 E/S/G 영역이 아니라 **정량/DART공시(자동) vs 정성 내부정책(HITL)**로 갈린다. 특히 G는 DART 정기공시 의무항목이라 자동화가 잘 됨(사외이사 비율·이사회 출석률·배당성향·주총·감사기구). E만 하면 이 공짜 win을 버리고, 폼도 반쪽이 됨.

**3. "자동응답 불가"도 정식 결과.** 전 28항목을 출력에 남기되 티어로 라우팅:
- 정량 + 공시 존재형 → `verified` / `self_reported` (+증빙 링크)
- 정성, 증빙만 올리면 풀림 → `insufficient` (+구체적 증빙 요청)
- 증빙 있어도 사람이 서술해야 함 → `hitl_required` (신규)
- 해당 없음 → `not_applicable` (신규, 선택)

## 작업 순서 — 수직 슬라이스 먼저 (수평 카탈로그 먼저 금지)

**STEP 1 — status 모델 확장**
- `schema.py` `AnswerStatus`에 `hitl_required` 추가 (필요시 `not_applicable`도). 의미 구분 명확히: insufficient(증빙 올리면 풀림) ≠ hitl_required(사람이 서술).
- 배지 매핑(`_BADGE`)에 신규 status 추가.
- 집계를 단일 `coverage_pct` → **3분할**(자동응답% / 사람필요% / 증빙대기%)로. mapping→gating→exporter→UI까지 일관 적용.

**STEP 2 — 28항목 → 증빙요구 매핑 테이블 (룩업, NLP 아님 → 싸다)**
- 각 정성 항목별로 "어떤 문서를 올리면 풀리는지" 선언형 테이블. 예: S-5-1 → 인권정책서, S-8-1 → 정보보호 정책/인증서, E-1-1 → 환경경영 방침서.
- 데이터 파일로 분리 추천: `data/frameworks/` 또는 `kesg_evidence_requirements.py`.

**STEP 3 — derive를 데이터타입 기준으로 분류 (derive_key 핸들러는 얇게)**
- 정량(numeric+D1) / 공시존재형 / 정성정책(→HITL/insufficient) 3갈래.
- 핸들러는 기존 신호(mapped, data_points, evidence_graph)를 **조합만**. 새 NLP/LLM 검출 넣지 말 것(= 두 번째 검출엔진 금지). 순수함수로 테스트 가능하게.
- 디폴트 핸들러 = 현재 presence/numeric/multi 로직 유지(하위호환).

**STEP 4 — 수직 슬라이스 1개 관통**
- 제일 안 풀리는 섹션 하나(예: 인권/노동 또는 안전보건)를 골라 전 경로 끝까지: derive → gating → hitl/insufficient → 증빙 체크리스트 → exporter(excel) → UI 탭.
- 이게 증명되면 그때 STEP 5.

**STEP 5 — 수평 확장: K-ESG 28 전체 카탈로그**
- K-ESG 28 자가진단 리포트를 1차 출력으로 시드(프레임워크 독립, 재사용 최대).
- SAQ/현대차 폼은 같은 캐노니컬 위 어댑터로 *그 다음에*.

**STEP 6 — 회귀 테스트**
- 실제 협력사 샘플 1개로 전체 회귀. 빈 섹션 없음 / 각 문항이 5개 status 중 하나로 귀결 / 엑셀·PDF 출력 누락 없음.
